Question: Bootstrap 3.1.1 Navbar Collapse not Working on iPhone 5s

I am trying to get the Bootstrap 3.1.1 Navbar collapse button to work on my iPhone 5s, and I'm pretty sure I'm doing it right, but I'm not getting the correct behavior.

I have a navbar with 2 rows, and a button in the last row which should collapse when you click the hamburger in the top left corner.

The problem is that the hamburger icon is not clickable and doesn't even collapse the menu. If i click anywhere on the menu it will collapse.

I also tried to put the menu on the top and the collapsible button on the left, but I got a white background on the menu.

What am I missing?

Answer: Bootstrap uses \strong{flexbox} by default. On the iPhone, it is \strong{not} supported yet.

See more here http://caniuse.com/#feat=flexbox

You can use `position:fixed;top:0;left:0;` for the button on an iPhone

Answer: You can use a media query like this:

\begin{code}
@media screen and (max-width: 768px) {
 .btn-navbar {
    float: none;
    margin-left: 15px
  }
}
\end{code}

Answer: Include this in your css file to solve problem.

\begin{code}
@media (max-width: 767px) {
   .navbar-default.collapse {
        display: none;
    }
}
@media (min-width: 768px) {
   .navbar-default.collapse {
        display: block;
    }
}
\end{code}
